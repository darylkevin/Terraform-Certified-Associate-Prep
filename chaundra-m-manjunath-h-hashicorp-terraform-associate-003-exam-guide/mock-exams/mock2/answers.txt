### Question 38

**Correct Answer(s):** A, B, D, E

**Options:**
- [✓] A. terraform apply -var="VARIABLE_NAME=VARIABLE_VALUE".
- [✓] B. If the variable and value are stored in the dev.tfvars file in the same working directory, run terraform apply -var-file="dev.tfvars".
- [ ] C. If the variable and value are stored in the dev.tfvars file in the same working directory, run terraform apply -variable-file="dev.tfvars".

- [ ] F. Export VAR_variable_name=variable-value.

**Explanation:**

Options A, B, D, and E are correct. Options C and F have the wrong syntax. There is a certain precedence in which variable values are picked by Terraform. Terraform loads variables in the following order, with later sources taking precedence over earlier ones: Any -var and -var-file options on the command line in the order they are provided. (This includes variables set by an HCP Terraform workspace.) Any *.auto.tfvars or *.auto.tfvars.json files, processed in alphabetical order by filename. The terraform.tfvars.json file, if present. The terraform.tfvars file, if present. Environment variables.
